Default Style

The default button is a secondary, clear, and regular button. This means that it has no background, border, background image, or inner box-shadows styles by default. Visually it looks like a link. Since the visual styling is so minimal, it’s considered a secondary button type.

Every button has :hover, :active, .is-selected :focus, and disabled states.

When using a button or an a element, be sure to specify a type or role, respectively.

Options

Depending on the action and the visual importance you would like it to receive in the UI, different classes are combined to create escalating levels of visual importance.

Primary

This style visually communicates the most important action on the screen. In practice you should declare this no more than once per a page. It is the most visually strong button style we use. The primary button does not have an outlined or clear style

Our buttons assume an icon size of 18px.

Secondary

Any button which is not a primary, confirm, or warning type is considered a secondary button. The secondary status refers to the visual importance we give the buttons. There are three states for a secondary button: a filled state, an outlined state, and a clear state. The clear state is the default style for all buttons.

The filled state is a muted version of .s-btn__primary. Use the filled style alongside the primary style to help communicate an important, but not as important, action. You should also use this when an action is the main action within a UI group, but none of the UI actions are considered a primary action for the page.

Confirm

The confirm button type can act like a primary action, but secondary styles are provided for it as well. It’s used primarily for final points in the workflow to communicate accomplishments. Examples would completing a purchase (checkout) or publishing a new item which is done rarely. The confim style is available in three states: filled, outlined, and clear. Clear is the default state.

{% highlight html linenos %} {% endhighlight %}

Warning

The warning button type can act like a primary action, but secondary styles are provided for it as well. It’s to be used to warn users about a potential negative action such as removing a post, removing a team member, or deleting their account. The warning style is available in three states: filled, outlined, and clear. Clear is the default state.

{% highlight html linenos %} {% endhighlight %}

Muted

The muted button style is a grayscale visual treatment to the secondary visual states. It creates a light gray button style. Typically this is used in layouts with multiple UI actions which are attempting to communicate settings which are turned off. This style is different from a disabled state. It is available in three states: filled, outlined, and clear. Clear is the default state.

Inverted

Most of our buttons appear on top of light background colors; yet there are moments where buttons must be placed on colorful or darker background colors. In these instances, apply the .s-btn__inverted class to invert the button styling.

Ordering

To maintain product consistency, buttons should maintain the following layout ordering:

Within a row

Most button groups should be ordered from the most important to the least important action, left to right.

{% highlight html linenos %} {% endhighlight %}

Within a column

Sometimes the layout dictates that buttons need to be stacked on top of each other. Again, these buttons should be stacked from the most important to the least important, top to bottom.

{% highlight html linenos %}
{% endhighlight %}

Within a header

Sometimes the best place for a series of actions is in the same area as the title. In these cases, the buttons should be pulled to the right. Within this instance, the button order should be reversed with the most important action to the far right and the least important action to the far left.
